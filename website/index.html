How it works

Three steps to total recall

Replay captures, understands, and indexes your terminal history so you can search it like a conversation.

01

Capture

Reads your shell history from Atuin — every command, timestamp, exit code, and directory. No output captured, just commands.

02

Understand

Embeds each command with Jina AI, clusters into sessions, detects fix patterns (fail then succeed), and filters secrets.

03

Recall

Indexes with FAISS for instant cosine-similarity search. Ask in natural language, get ranked results with full context.

Features

Built for developers who forget

🔍

Semantic search

Ask in plain English. "How did I fix that nginx config?" returns the exact command sequence, ranked by meaning.

🔧

Fix detection

Automatically finds failure-then-success patterns. See how you solved problems before, not just what you ran.

📁

Session clustering

Commands are grouped by session, directory, and time. Context before and after every result so you see the full picture.

🔒

Secret filtering

16 regex patterns catch API keys, tokens, and passwords before they reach the embedding API. Your secrets stay local.

⚡

Instant results

FAISS inner-product search over normalized vectors. Sub-second queries across thousands of commands.

📦

Zero config

One command to install, one to search. Auto-detects Atuin DB, builds index on first run. Works with any shell.

Get started

Install in seconds

Replay is a Python CLI tool. Install via pip, then search your history.

Quick start

$ pip install replay-ai
$ replay init
$ replay search "docker fix"

Requires Atuin for shell history capture. Embeddings via Jina AI (free tier).
