Capabilities · Nº 03

Four pillars of terminal intelligence.

From capturing raw Atuin history to serving instant semantic results — every layer is open-source, local-first, and free to run.

01Capture

Atuin
Capture

Reads every command, timestamp, exit status, directory, and session from Atuin's SQLite history — no output stored, privacy by default.

02Embed

Jina AI
Embeddings

Every command chunk embedded with Jina v3 (1024 dims) — free tier, no API key cost. Batch processing with exponential backoff retry.

03Index

FAISS Vector
Search

Cosine similarity over L2-normalized embeddings. Sub-millisecond search over thousands of commands. Auto-builds on first query.

04Detect

Fix
Detection

Automatically finds failure-then-success patterns: docker build fails, sudo docker build succeeds. Your personal debugging playbook.

V. Method / Pipeline • 04 stages, semantic 005 / 008
How it works · Nº 05

From raw commands to semantic memory.

+

Every stage runs locally. No cloud. No telemetry. Your commands stay on your machine.

01

Capture →

Atuin records every command with timestamp, exit status, directory, and session. Replay reads the SQLite DB directly — read-only, zero overhead.

02

Process →

Commands are clustered into sessions (by Atuin session ID or 15-min heuristic gaps). Trivial commands (cd, ls, pwd) become context, not primary chunks.

03

Embed →

Each chunk gets embedded with Jina AI v3 (1024 dimensions, free tier). Secrets are automatically redacted before hitting the API — 16 regex patterns cover keys, tokens, and passwords.

04

Search

Natural language queries are embedded and matched via FAISS cosine similarity. Results include the command, exit status, directory, and surrounding context.

Everything runs locally. Your data never leaves your machine.

Fix Detection 01 / 02

Failure → Success Patterns

Tracks consecutive failures within a session. When a success follows, it records the fix. 'docker build' fails, 'sudo docker build' succeeds — Replay remembers the fix for next time.
